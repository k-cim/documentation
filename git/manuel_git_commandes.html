🛠️ Manuel Git – Commandes principales
Version 0.01.00

Ce manuel présente les principales commandes Git utilisées en ligne de commande pour initialiser, cloner, versionner, pousser, récupérer, et gérer l’historique d’un dépôt.

📁 1. Initialisation d’un dépôt

git init

Initialise un dépôt Git vide dans le dossier courant.

🌐 2. Clonage d’un dépôt distant

git clone URL

Crée une copie locale d’un dépôt distant.

📌 3. Vérifier le statut du dépôt

git status

Affiche les fichiers modifiés, ajoutés, ou en attente de commit.

➕ 4. Ajouter des fichiers à l’index

git add fichier

Prépare les fichiers à être commités.

📝 5. Commit des modifications

git commit -m "Message explicite"

Valide les changements ajoutés avec un message.

🚀 6. Pousser les changements sur le dépôt distant

git push origin branche

Envoie les commits locaux sur GitHub ou autre remote.

📥 7. Récupérer les changements distants

git pull

Récupère et fusionne les modifications depuis le dépôt distant.

🔄 8. Gestion des branches

git branch

Affiche les branches locales.

git branch nouvelle-branche

Crée une nouvelle branche.

git checkout branche

Bascule sur une autre branche.

git merge branche

Fusionne une branche dans celle courante.

🧹 9. Suppression d’une branche

git branch -d branche

Supprime une branche locale.

🔍 10. Historique des commits

git log

Affiche la liste des commits.

git log --oneline --graph --all

Affichage condensé avec branches visibles.

🎯 11. Utilisation des tags

git tag v1.0.0

Crée un tag de version.

git push origin --tags

Envoie les tags vers le dépôt distant.

♻️ 12. Revenir à une version précédente

git checkout commit_id -- chemin/fichier

Restaure un fichier à partir d’un commit antérieur.

💡 Astuce : Utilisez git help pour obtenir l’aide spécifique à chaque commande.

Consultez aussi la documentation officielle sur git-scm.com/doc.
