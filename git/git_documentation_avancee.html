📖 Chapitre 1 : Introduction à Git

Git est un système de contrôle de version distribué largement utilisé dans le développement logiciel. Il permet aux développeurs de collaborer efficacement et de gérer facilement les historiques de leurs projets.

🛠️ Chapitre 2 : Installation et Configuration

Installation sous Linux

sudo apt update
sudo apt install git

Configuration initiale

git config --global user.name "Votre Nom"
git config --global user.email "votre@email.com"
git config --global core.editor "vim"

Fichiers de configuration

  • ~/.gitconfig – Configurations globales de l'utilisateur
  • .git/config – Configurations spécifiques au dépôt

🔑 Chapitre 3 : Gestion des clés SSH

La connexion aux dépôts distants sécurisés se fait via des clés SSH :

ssh-keygen -t ed25519 -C "votre@email.com"
eval "$(ssh-agent -s)"
ssh-add ~/.ssh/id_ed25519
cat ~/.ssh/id_ed25519.pub

🚦 Chapitre 4 : Gestion des dépôts

Créer et cloner un dépôt

git init nom_du_projet
git clone git@github.com:utilisateur/projet.git

Ajouter et suivre des fichiers

git add fichier.txt
git commit -m "Message descriptif"

🌳 Chapitre 5 : Branches et Fusions

Gestion des branches

git branch nouvelle_branche
git checkout nouvelle_branche
git merge branche_source

🔍 Chapitre 6 : Inspection et débogage

Logs avancés

git log --graph --oneline --decorate --all

Historique des modifications

git blame fichier.txt

⚡ Chapitre 7 : Utilisation avancée

Alias Git personnalisés

git config --global alias.st "status"
git config --global alias.co "checkout"
git config --global alias.br "branch"

Hooks Git

Automatisation de tâches avant ou après certains événements :

.git/hooks/pre-commit
.git/hooks/post-commit

🚨 Chapitre 8 : Gestion des erreurs courantes

  • Résolution de conflits : git mergetool
  • Annulation d'un commit : git revert [commit]
  • Modification du dernier commit : git commit --amend
